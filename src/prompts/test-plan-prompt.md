# Test Plan Creation Prompt

## Purpose

This prompt can be used to generate a professional QA test plan using AI. The QA Engineer should provide the project details, module details, testing scope, tools, environment, timeline, and risks before generating the final test plan.

## Prompt

Act as a Senior QA Engineer and create a complete professional test plan for the following software project.

## Project Details

Project Name: [Enter project name]

Module / Feature: [Enter module or feature name]

Project Description: [Briefly explain the project or feature]

Requirement / User Story: [Paste the requirement or user story here]

Testing Type: [Functional Testing, Regression Testing, UI Testing, API Testing, Smoke Testing, etc.]

Testing Environment: [QA / UAT / Staging / Production]

Tools Used: [Jira, Postman, Selenium, TestRail, Excel, GitHub, etc.]

Timeline: [Enter testing timeline]

QA Team: [Enter QA team details]

Developer Team: [Enter developer team details if available]

## Create the Test Plan With the Following Sections

1. Test Plan ID
2. Project Name
3. Module / Feature Name
4. Objective
5. Scope of Testing
6. Out of Scope
7. Test Approach
8. Testing Types
9. Test Environment
10. Test Data Requirements
11. Entry Criteria
12. Exit Criteria
13. Test Deliverables
14. Risks and Mitigation
15. Roles and Responsibilities
16. Test Schedule
17. Defect Management Process
18. Assumptions
19. Dependencies
20. Approval / Sign-Off

## Instructions

Create the test plan in a professional QA document format.

Use clear headings and tables where suitable.

Include practical QA details, not only general explanations.

Mention assumptions clearly if any requirement details are missing.

Identify possible risks and provide suitable mitigation actions.

Keep the language clear, formal, and suitable for adding to a GitHub QA portfolio.
